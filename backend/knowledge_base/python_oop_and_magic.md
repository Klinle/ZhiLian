# Python 面向对象高级编程与魔术方法

## 1. 类、实例与初始化 (Classes & Instances)
*   **实例化流程**：当调用 `MyClass()` 实例化对象时，Python 解释器先执行 `__new__(cls)` 方法在内存中分配空间并创建空白实例，接着调用 `__init__(self)` 传入该实例进行状态初始化。
*   **self 含义**：`self` 只是局部变量名，表示被创建的实例本身。所有实例方法的第一个参数必须是它，用来绑定当前操作的对象数据。

## 2. 继承、多态与 MRO 拓扑
*   **多继承与 MRO**：Python 支持一个子类继承多个父类。当寻找一个方法时，Python 按照 **MRO (Method Resolution Order, 方法解析顺序)** 线性列表进行检索。
*   **C3 算法**：CPython 使用 C3 线性化算法计算多继承拓扑，确保单调性（子类优先于父类）和局部优先级。你可以通过 `MyClass.__mro__` 打印检索顺序。使用 `super().method()` 调用时，是按照 MRO 拓扑链条的下一个类进行查找，而非单纯调用直接父类。

## 3. 魔术方法重载 (Dunder Methods)
魔术方法（双下划线方法）允许自定义对象接入 Python 的内置协议：
*   `__add__(self, other)`：重载加号 `+`。当执行 `a + b` 时，自动调用 `a.__add__(b)`。
*   `__str__(self)` 与 `__repr__(self)`：分别定义易读的打印字符串表示（`print()` 触发）与精确的开发者调试表示（控制台输入触发）。
*   `__call__(self)`：使实例可以像函数一样被括号调用，即 `obj()` 自动执行 `obj.__call__()`。

## 4. 属性管理与 slots 优化
*   **属性拦截 (Property)**：使用 `@property` 装饰器可以将一个方法伪装成属性进行读取，通过对应的 `.setter` 方法拦截写入逻辑，从而在写入前执行安全范围检查。
*   **__slots__ 优化**：默认情况下，Python 实例使用一个动态字典 `__dict__` 存储所有实例属性，这会消耗大量内存。通过在类中声明 `__slots__ = ['name', 'age']`，Python 会在内存中使用紧凑的固定长度数组代替字典，大幅节省数十万级量产对象时的内存消耗，但代价是禁止动态添加 slots 列表外的属性。
