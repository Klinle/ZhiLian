# Python 基础语法与核心机制权威参考

## 1. 变量与动态类型 (Variables & Dynamic Typing)
Python 采用的是**“命名绑定”**的变量模型，而不是传统的“内存值拷贝”模型：
*   **引用机制**：变量名本质上是一个指向内存中对象的“标签（指针）”。例如 `x = 10` 表示在内存中创建了一个整数对象 `10`，并在当前的局部/全局名字空间（Namespace）里，把标签 `x` 贴在了它上面。
*   **动态类型**：变量本身没有类型，类型属于内存中的对象。因此 `x` 可以先绑定整数 `x = 10`，随后再绑定字符串 `x = "Python"`。
*   **小整数对象池**：为了提升效率，CPython 虚拟机在启动时会自动预创建并缓存 `[-5, 256]` 范围内的整数对象。在这个区间内的任何变量赋值都会复用同一个对象引用，例如 `a = 256; b = 256` 时 `a is b` 返回 `True`；而当值为 257 时则会创建两个独立的实例。

## 2. 内置容器与操作 (Built-in Containers)
Python 提供了四大核心内置容器，它们在内存结构与查找效率上各有绝活：
*   **列表 (List)**：底层为**动态紧凑数组**。支持 $O(1)$ 的随机索引访问，但在数组中间执行插入或删除操作会引发大量元素的内存拷贝移动，复杂度为 $O(n)$。
*   **元组 (Tuple)**：**不可变列表**。在创建后其引用的对象不可替换，内存占用比列表略小，常用于返回多值或作为字典的 Key（因为其哈希值不可变）。
*   **字典 (Dict)**：底层基于 **哈希表 (Hash Table)** 实现。当查询一个键时，Python 计算键的 `hash()` 值，映射到数组下标实现极速定位，查询、插入、删除的平均复杂度均为 $O(1)$。
*   **集合 (Set)**：基于哈希表实现的无序非重叠容器，只有 Key 没有 Value，天然适用于数据去重与高速集合交并差运算。

## 3. 控制流与函数解包 (Control Flow & Parameter Unpacking)
*   **控制流**：Python 依赖严格的 **4空格缩进** 代替大括号来划分作用域。`if-elif-else` 进行逻辑分支选择，`for...in` 用于遍历可迭代对象。
*   **参数解包**：
    *   `*args` 用于接收任意数量的位置参数，函数内部将其打包为元组；
    *   `**kwargs` 用于接收任意数量的关键字参数，函数内部将其打包为字典；
    *   在函数调用时，`func(*list)` 和 `func(**dict)` 则用于将容器解包分发给对应的形参。
