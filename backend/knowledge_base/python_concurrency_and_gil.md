# Python 并发编程与全局解释器锁 (GIL)

## 1. GIL 全局解释器锁 (Global Interpreter Lock)
*   **核心机制**：CPython 解释器的内存管理不是线程安全的。为了防止多个线程在并发修改 Python 对象时发生引用计数混乱，CPython 在虚拟机中引入了全局锁 GIL。
*   **物理铁律**：在同一时间，GIL 保证只有一个线程能获得 CPython 解释器的执行权。因此，即使你的 CPU 拥有 16 个核心，Python 的多线程代码在 CPU 密集计算时也只能利用 1 个核心，无法实现真正的并行。

## 2. 多线程与多进程的抉择
*   **多线程 (Threading)**：由操作系统调度。在执行网络请求、数据库查询、磁盘读写等 I/O 操作时，当前线程会主动释放 GIL 锁，让其他线程运行。因此，**多线程非常适用于 I/O 密集型任务（如爬虫、聊天室）**。
*   **多进程 (Multiprocessing)**：每个子进程拥有独立的 Python 解释器、独立的内存空间和独立的 GIL 锁。因此，**多进程可以利用多核 CPU 进行并行计算，特别适用于 CPU 密集型任务（如物理渲染、矩阵变换、加密计算）**。进程间通信（IPC）需要使用 Queue 或 Pipe 进行数据序列化传输。

## 3. 异步协程 (asyncio) 核心原理
*   **非阻塞主循环**：`asyncio` 是一种**协作式单线程异步编程框架**。它在底层运行一个事件循环（Event Loop），不断清点就绪的任务。
*   **工作机制**：通过 `async def` 定义协程，在遇到网络 I/O 阻碍时使用 `await` 主动出让 CPU 执行权。事件循环会立即安排其他任务执行，直至 I/O 数据返回后再切回继续。这避免了线程上下文切换的昂贵开销，能够用**单核单线程处理数十万的高并发网络连接**。
